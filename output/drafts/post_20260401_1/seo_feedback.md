## SEO 분석 보고서

### 타겟 키워드 제안

- **메인 키워드**: 데이터 준비도 (data readiness)
- **서브 키워드**: AI 도입 실패 원인, 기업 데이터 정비, 데이터 사일로, AX 도입 준비, AI 프로젝트 실패

---

## 제목 최적화

**현재 제목**: `AI 도입 전에 먼저 물어야 할 것 — "우리 데이터, 쓸 수 있는 상태인가?"`

**평가**:
- 길이: 공백 포함 약 41자 — 한글 기준 30~60자 범위 내. 적절함.
- 질문형 구조로 클릭 유도 요소는 있음.
- 그러나 메인 키워드 "데이터 준비도"가 제목에 등장하지 않음. 검색 유입 측면에서 취약.
- 부제("우리 데이터, 쓸 수 있는 상태인가?")가 구어체라 브랜드 보이스에 맞고 독자 공감을 끌지만, 검색 결과에서 키워드 매칭이 약함.
- "먼저 물어야 할 것"은 다소 모호하여 검색 의도와 직접 연결되지 않음.

**대안 제안**:

| 안 | 제목 | 특징 |
|---|---|---|
| A | `AI 도입 전 데이터 준비도 점검법 — 4가지 축으로 진단하기` | 메인 키워드 전방 배치 + 숫자 포함, SEO 최적화에 유리 |
| B | `AI 프로젝트 95%가 실패하는 진짜 이유 — 데이터 준비도 4단계 점검` | 충격적 수치로 클릭 유도 + 메인 키워드 포함 |
| C | `"우리 데이터, AI에 쓸 수 있나요?" — 기업 데이터 준비도 자가 진단 가이드` | 브랜드 보이스 유지 + 메인 키워드 포함, 소셜 공유에도 적합 |

**권고**: 안 B 또는 C. Thought Leadership 카테고리 특성상 소셜 공유도 고려해야 하므로, 수치 클릭 유도(B)와 독자 질문 공감(C)을 결합하는 방향이 이상적. 현재 제목의 대시(—) 구분 형식은 유지 가능.

## 메타 디스크립션

**현재 메타 디스크립션**: `AI 프로젝트 95%가 임팩트를 내지 못하는 원인은 모델이 아니라 데이터입니다. AX를 시작하기 전에 우리 회사 데이터가 실제로 쓸 수 있는 상태인지 점검하는 4가지 프레임워크를 공유합니다.`

**글자 수**: 약 91자 (한글 기준)

**평가**:
- 길이: 91자로 체크리스트 기준(80~120자) 범위 내. 적절함.
- "AI 프로젝트 95%"라는 수치로 클릭 유도력이 있음. 강점.
- 타겟 키워드 "데이터 준비도"가 명시적으로 포함되지 않음. "4가지 프레임워크"는 포함됨.
- 행동 유도 문구 부재. "점검하는 4가지 프레임워크를 공유합니다"는 다소 수동적.

**개선안**:

> `AI 프로젝트 95%가 실패하는 진짜 원인은 모델이 아니라 데이터 준비도입니다. 품질·사일로·거버넌스·드리프트 4가지 축으로 우리 회사 데이터를 지금 바로 진단해보세요.` (87자)

- "데이터 준비도" 메인 키워드 명시
- 4가지 축 키워드 직접 노출 → 검색 스니펫에서 세부 항목이 노출될 가능성 증가
- "지금 바로 진단해보세요" 행동 유도 문구 추가

## 헤딩 구조 피드백

**현재 헤딩 구조**:
- H1: `AI 도입 전에 먼저 물어야 할 것 — "우리 데이터, 쓸 수 있는 상태인가?"`
- H2: `왜 AI 프로젝트 95%가 임팩트를 내지 못하는가`
- H2: `데이터 준비도를 가르는 4가지 축`
- H3: `1축 — 품질: 왜 AI는 좋은 데이터 없이 좋은 답을 못 낼까?`
- H3: `2축 — 사일로: 섬처럼 나뉜 데이터는 AI에게도 섬이다`
- H3: `3축 — 거버넌스: 처음부터 설계하지 않으면, 나중에 두 배 비용이 든다`
- H3: `4축 — 드리프트: AI가 조용히 틀리기 시작할 때`
- H2: `데이터 정비가 AX를 가속한다`
- H2: `막막하다면, 오늘 여기서 시작하세요`

**평가**:

| 현재 헤딩 | 평가 | 제안 헤딩 | 이유 |
|---|---|---|---|
| `왜 AI 프로젝트 95%가 임팩트를 내지 못하는가` | 양호. 수치 포함, 독자 관심 유도. "AI 도입 실패 원인" 키워드 포함 가능 | `AI 프로젝트 95%가 실패하는 원인 — 데이터 준비도의 민낯` | "데이터 준비도" 메인 키워드 연결 강화 |
| `데이터 준비도를 가르는 4가지 축` | 우수. 메인 키워드 "데이터 준비도" 포함. 유지 권고. | 유지 | — |
| `1축 — 품질: 왜 AI는 좋은 데이터 없이 좋은 답을 못 낼까?` | 양호. 브랜드 보이스 맞음. "데이터 품질" 키워드 포함 가능. | `1축 — 데이터 품질: AI가 좋은 답을 내려면 좋은 재료가 있어야 한다` | "데이터 품질" 검색 용어 직접 노출 |
| `2축 — 사일로: 섬처럼 나뉜 데이터는 AI에게도 섬이다` | 양호. 비유가 강렬하고 브랜드 보이스에 부합. "데이터 사일로" 키워드 포함 우선. | `2축 — 데이터 사일로: 부서 간 데이터 단절이 AI를 망친다` | "데이터 사일로" 검색 노출 강화 |
| `3축 — 거버넌스: 처음부터 설계하지 않으면, 나중에 두 배 비용이 든다` | 양호. 비용 언급으로 임팩트 있음. "데이터 거버넌스" 키워드 앞배치 권고. | `3축 — 데이터 거버넌스: 나중에 잡으면 두 배 비용이 든다` | "데이터 거버넌스" 검색 용어 앞배치 |
| `4축 — 드리프트: AI가 조용히 틀리기 시작할 때` | 양호. 브랜드 보이스 강함. "모델 드리프트" 키워드 포함 가능. | `4축 — 데이터 드리프트: AI가 조용히 현실을 놓치는 순간` | "데이터 드리프트" 검색 용어 연결 |
| `데이터 정비가 AX를 가속한다` | 보통. "데이터 정비" + "AX 도입" 키워드 모두 유효. 유지 가능. | 유지 | — |
| `막막하다면, 오늘 여기서 시작하세요` | 브랜드 보이스 우수. SEO 측면에서 키워드 미포함. CTA 성격이 강하여 SEO 소제목으로는 취약. | `데이터 준비도 자가 점검: 오늘 바로 시작하는 3가지 방법` | "데이터 준비도 자가 점검" 검색 유입 가능, 숫자(3가지) 포함 |

**종합 평가**:
- H1은 1개로 올바르게 사용됨.
- H2 4개, H3 4개로 계층 구조 준수. H3는 H2(`데이터 준비도를 가르는 4가지 축`) 하위에 있어 적절.
- 핵심 개선 포인트: H3 소제목 전반에서 "데이터 품질", "데이터 사일로", "데이터 거버넌스", "데이터 드리프트" 등 검색 가능한 용어를 앞쪽에 배치하면 서브 키워드 검색 유입이 증가함.

## 본문 SEO 피드백

### 키워드 배치 현황

| 위치 | 현재 상태 | 권고사항 |
|---|---|---|
| 제목(H1) | "데이터" 포함, "데이터 준비도" 미포함 | "데이터 준비도" 메인 키워드 명시 권고 |
| 첫 문단 | "데이터", "AI 도입" 자연스럽게 등장. "데이터 준비도" 미포함 | 세 번째 문단("바로, 데이터가 실제로 쓸 수 있는 상태인가")에 "데이터 준비도" 용어 1회 자연 삽입 권고 |
| H2 소제목 | "데이터 준비도를 가르는 4가지 축" — 메인 키워드 포함. 우수. | 유지 |
| 본문 전체 | "데이터 준비도" 표현이 2회(H2, 실행포인트 안내문) 등장. 다소 적음. | 3~4회 수준으로 자연스럽게 증가 권고. 과최적화 주의. |
| 서브 키워드 | "데이터 사일로", "데이터 거버넌스", "드리프트" 모두 본문 내 정의와 함께 등장. 우수. | 유지 |

### 키워드 밀도 분석

- 전체 본문 약 1,700자(한글 기준 추정)로 최소 기준(1,000자) 충족. 권장 기준(2,000자 이상)에는 약간 미달. 분량 자체는 실무형 글로서 적절한 편.
- "데이터" 단어 빈도: 매우 높음. 자연스러운 반복이나 일부 "정보", "자산" 등 동의어로 변주 고려.
- "AI 도입", "AX" 키워드는 고르게 분포됨. 적절함.

### 내부 링크 기회

- 본문에서 "PBT VLM OCR Suite" 제품명 언급이 있으나 제품 페이지 링크가 없음. 내부 링크 삽입 권고.
- "매직에꼴 AX 컨설팅" CTA 링크는 있으나, 관련 블로그 글(예: AI 에이전트 트렌드, RAG 활용 사례 등)로의 내부 링크가 전무함. 관련 글 2~3편 연결 권고.
- 내부 링크 앵커 텍스트 예시: "AI 에이전트 도입 사례 보기", "RAG 기반 지식 관리 방법" 등 서술형 사용 권고.

### 이미지 alt 텍스트

- 현재 초안에 이미지 삽입 마크다운이 없음(이미지 생성 전 단계). 이미지 삽입 시 다음 규칙 적용 권고:
  - 썸네일: `alt="기업 데이터 준비도 점검 — AI 도입 전 체크리스트"`
  - 일러스트 1(데이터 준비도 4축 표 관련): `alt="데이터 준비도 4가지 축 — 품질, 사일로, 거버넌스, 드리프트"`
  - 일러스트 2: `alt="AI 프로젝트 실패 원인 — 데이터 준비도 통계"`
  - 다이어그램(있을 경우): `alt="기업 데이터 준비도 점검 프레임워크 다이어그램"`

### URL 구조 제안

- 영문 슬러그: `ai-data-readiness-checklist` 또는 `enterprise-data-readiness-ai`
- 한글 슬러그: `AI-도입-데이터-준비도-점검` (플랫폼이 한글 지원 시)

## 우선순위 수정 항목

1. **[긴급] 제목에 "데이터 준비도" 메인 키워드 포함**: 현재 제목은 브랜드 보이스는 우수하나 메인 키워드가 없어 검색 유입에서 약함. 안 B 또는 C로 교체 권고.
2. **[중요] 메타 디스크립션 개선**: "데이터 준비도" 키워드 명시 + 행동 유도 문구("지금 바로 진단해보세요") 추가. 개선안으로 교체 권고.
3. **[권고] H3 소제목 키워드 앞배치**: "데이터 품질", "데이터 사일로", "데이터 거버넌스", "데이터 드리프트" 용어를 각 H3 앞쪽에 배치. 브랜드 보이스 훼손 없이 수정 가능.
4. **[권고] 첫 문단 키워드 삽입**: 3번째 문단에서 "데이터가 실제로 쓸 수 있는 상태인가" → "데이터 준비도(data readiness)가 충분한가" 형태로 1회 자연 삽입.
5. **[선택] 내부 링크 추가**: PBT VLM OCR Suite 제품 페이지 + 관련 블로그 글 2편 내부 링크 삽입. 독자 체류 시간 및 페이지 권위 향상.
