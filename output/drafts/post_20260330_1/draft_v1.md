# RAG가 뭔가요? — 기본 RAG부터 GraphRAG까지, AI가 정보를 찾는 방식의 진화

ChatGPT에게 우리 회사 내부 문서를 물어보면 어떻게 될까요? 아마 그럴듯한 답을 내놓겠지만, 절반은 지어낸 이야기일 겁니다. AI가 학습하지 않은 정보는 답할 수 없으니까요. 이 문제를 해결하려고 등장한 기술이 RAG입니다. 그런데 RAG에도 종류가 있습니다. 어떤 RAG를 쓰느냐에 따라 AI의 답변 품질이 완전히 달라집니다.

## RAG, 한마디로 뭔가요?

RAG는 Retrieval-Augmented Generation의 약자입니다. 한국어로 옮기면 "검색 증강 생성" 정도 됩니다. 이름이 어렵지만, 원리는 단순합니다.

AI에게 질문하기 전에, 먼저 관련 문서를 찾아서 함께 건네주는 것입니다.

도서관에 비유하면 이렇습니다. 일반 AI는 자기 머릿속 기억만으로 답합니다. 반면 RAG는 질문을 받으면 도서관 서가에서 관련 책을 먼저 꺼내 읽고, 그 내용을 바탕으로 답합니다. 당연히 후자가 더 정확하겠죠.

기업에서 RAG가 중요한 이유도 여기에 있습니다. 사내 문서, 매뉴얼, 계약서처럼 AI가 학습한 적 없는 비공개 데이터를 활용할 수 있게 해주기 때문입니다.

---

## 기본 RAG — 가장 단순한 형태

가장 기초적인 RAG를 Baseline RAG 또는 Naive RAG라고 부릅니다. 작동 방식은 세 단계입니다.

1. **문서 쪼개기** — 사내 문서를 작은 조각(chunk)으로 나눕니다.
2. **벡터로 변환** — 각 조각을 숫자 배열(벡터)로 바꿔서 저장합니다.
3. **유사도 검색** — 질문이 들어오면, 질문과 가장 비슷한 조각을 찾아 AI에게 건네줍니다.

벡터는 텍스트의 의미를 숫자로 압축한 것입니다. "사과"와 "과일"의 벡터는 가깝고, "사과"와 "자동차"의 벡터는 멉니다. 이 거리를 계산해서 관련 문서를 찾는 겁니다.

이 방식은 간단하고 빠릅니다. 특정 주제에 대한 구체적인 질문에는 꽤 잘 작동합니다. "A 제품의 보증 기간이 어떻게 되나요?" 같은 질문이 여기에 해당됩니다.

그런데 한계가 분명합니다.

> **핵심**: 기본 RAG는 "비슷한 문장 찾기"에 강하지만, 여러 문서에 흩어진 정보를 연결하거나 전체적인 맥락을 파악하는 데는 취약합니다.

예를 들어 "우리 회사 전체 매뉴얼의 주요 테마가 뭐야?"라고 물으면, 기본 RAG는 제대로 된 답을 내놓기 어렵습니다. 비슷한 문장 몇 개를 찾아올 뿐, 전체 그림을 그리지 못하기 때문이죠.

## Advanced RAG — 검색 품질을 높이는 방법

기본 RAG의 약점을 보완한 것이 Advanced RAG입니다. 핵심은 검색 전후에 추가 처리를 넣는 것입니다.

### 검색 전 — 더 똑똑하게 준비하기

문서를 쪼갤 때 단순히 글자 수로 자르지 않습니다. 문단, 섹션, 표 단위로 의미 있게 나눕니다. 질문도 그대로 검색하지 않고, 여러 방향으로 재구성해서 검색 범위를 넓힙니다.

### 검색 후 — 결과를 다시 걸러내기

검색 결과를 그대로 AI에게 넘기지 않습니다. 관련성이 높은 순서로 다시 정렬(Re-ranking)하고, 중복이나 노이즈를 제거한 뒤 전달합니다.

Advanced RAG는 기본 RAG보다 답변 품질이 높습니다. 하지만 근본적인 구조는 같습니다. 여전히 "비슷한 텍스트 조각 찾기"에 의존합니다. 정보 간의 관계를 이해하지는 못합니다.

---

## GraphRAG — 지식 그래프로 전체 그림을 읽다

GraphRAG는 접근 방식 자체가 다릅니다. 텍스트 조각이 아니라, 지식의 구조를 만들어서 검색합니다.

Microsoft Research가 2024년에 발표한 이 기술은 두 가지 단계로 작동합니다.

### 1단계: 지식 그래프 만들기

문서에서 사람, 조직, 개념 같은 핵심 요소(엔티티)를 추출하고, 이들 사이의 관계를 그래프로 연결합니다. 그런 다음 밀접하게 관련된 요소들을 그룹(커뮤니티)으로 묶고, 각 그룹의 요약을 미리 생성해둡니다.

쉽게 말해, 문서를 읽고 "누가 누구와 어떤 관계인지" 지도를 그리는 겁니다.

### 2단계: 질문에 답하기

질문이 들어오면, 관련된 커뮤니티 요약들을 활용해 부분 답변을 만들고, 이를 종합해 최종 답변을 생성합니다.

기본 RAG가 서가에서 비슷한 책을 찾아오는 사서라면, GraphRAG는 도서관 전체의 분류 체계를 이해하고 있는 학자에 가깝습니다. "이 도서관에서 가장 많이 다뤄지는 주제가 뭐야?"라는 질문에도 답할 수 있죠.

GraphRAG의 실제 효과도 확인되고 있습니다. Data.world는 GraphRAG로 LLM 응답 정확도가 평균 3배 향상됐다고 보고했습니다. LinkedIn은 고객 서비스 이슈당 중앙 해결 시간을 28.6% 단축했습니다. Microsoft 연구에서는 대안적 접근법 대비 26~97% 적은 토큰을 사용하면서도 더 나은 답변을 제공했습니다.

> **핵심**: GraphRAG는 단순 텍스트 유사도를 넘어, 데이터 전체의 구조와 맥락을 이해하는 방식입니다. 특히 "전체 그림"을 봐야 하는 질문에서 기존 RAG와 결정적인 차이를 만듭니다.

## 세 가지 RAG, 어떻게 다른가요?

여기까지 읽었다면, 핵심적인 차이가 보일 겁니다. 한눈에 정리하면 이렇습니다.

| 구분 | 기본 RAG | Advanced RAG | GraphRAG |
|---|---|---|---|
| 검색 방식 | 텍스트 유사도 | 텍스트 유사도 + 전후처리 | 지식 그래프 + 커뮤니티 요약 |
| 잘하는 질문 | 구체적인 사실 확인 | 구체적 질문 + 일부 복합 질문 | 전체 맥락, 관계 파악, 요약 |
| 못하는 질문 | 여러 문서 연결, 전체 요약 | 데이터 전체의 구조 파악 | 단순 사실 확인(과잉 설계) |
| 구축 난이도 | 낮음 | 중간 | 높음 |
| 비용 | 낮음 | 중간 | 초기 구축 높음, 운영 시 효율적 |
| 설명 가능성 | 낮음 | 낮음 | 높음 (그래프로 추적 가능) |

단순한 FAQ 챗봇이라면 기본 RAG로 충분합니다. 사내 문서 검색의 정확도를 높이고 싶다면 Advanced RAG가 적합합니다. 하지만 수백 개의 문서에서 전체적인 인사이트를 뽑아내야 하거나, 답변의 근거를 투명하게 추적해야 하는 상황이라면 GraphRAG가 필요합니다.

여러분의 조직에서 AI에게 던지는 질문은 어떤 유형인가요?

## GraphRAG가 특히 강한 세 가지 장면

### 장면 1: 대규모 문서에서 핵심 테마 찾기

수천 페이지 분량의 사내 보고서, 회의록, 이메일에서 "올해 가장 자주 언급된 이슈가 뭐지?"라고 물었을 때. 기본 RAG는 비슷한 문장 몇 개를 던져줄 뿐이지만, GraphRAG는 전체 데이터의 구조를 파악해 종합적인 답을 만들어냅니다.

### 장면 2: 흩어진 정보 연결하기

"A 고객사와 관련된 모든 프로젝트, 담당자, 이슈를 정리해줘." 이런 질문은 여러 문서에 흩어진 정보를 연결해야 합니다. 지식 그래프는 엔티티 간의 관계를 이미 매핑해두고 있어서, 점과 점을 잇는 작업에 강합니다.

### 장면 3: 감사와 규제 대응

금융, 헬스케어처럼 규제가 강한 산업에서는 "AI가 왜 이 답을 냈는가"를 설명할 수 있어야 합니다. GraphRAG는 지식 그래프를 통해 답변의 근거를 시각적으로 추적할 수 있습니다. 벡터 배열만으로는 불가능한 설명 가능성과 감사 가능성을 제공합니다.

## 어떤 RAG를 선택해야 할까요?

정답은 "상황에 따라 다르다"입니다. 세 가지 RAG는 경쟁 관계가 아니라 진화의 단계이면서, 동시에 용도가 다른 도구입니다.

지금 사내에 AI 챗봇을 도입하려 한다면, 기본 RAG부터 시작해도 됩니다. 단, 질문이 점점 복잡해지고 데이터가 쌓일수록, 검색 품질의 한계에 부딪히게 됩니다. 그때가 Advanced RAG나 GraphRAG로 확장할 시점입니다.

LlamaIndex 창업자 Jerry Liu가 지적한 것처럼, GraphRAG는 기존 RAG를 대체하는 것이 아니라 포함하는 개념입니다. 벡터 검색 위에 지식 그래프를 얹는 구조이기 때문에, 기본 RAG의 장점을 유지하면서 한계를 넘어서는 방식입니다.

준비된 조직은 이미 데이터를 그래프로 구조화하고 있습니다. 아직 RAG 자체가 낯설다면, 지금이 출발점을 잡을 때입니다. 어디서부터 시작해야 할지 막막하다면, 함께 설계하면 됩니다.
