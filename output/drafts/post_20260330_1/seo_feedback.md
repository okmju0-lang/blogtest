- "RAG가 뭔가요?"라는 질문형 도입부가 검색 의도(informational query)에 잘 부합함.

**권장 대안**:

2. "RAG 뜻과 종류 완벽 정리: 기본 RAG부터 GraphRAG까지" (30자) — "RAG 뜻" 검색어 포착
3. 현재 제목 유지하되, 부제를 "기본 RAG부터 GraphRAG까지 비교"로 축약 (36자)

**조치**: 제목 대안 중 하나를 채택하거나, 현재 제목의 부제를 축약하여 SERP 노출 최적화.

## 타겟 키워드 분석

### 현재 키워드 분포

| 키워드 | 출현 횟수 | 분포 평가 |
|---|---|---|
| RAG | 37회 | 충분 — 제목, 본문, 헤딩 전반에 고르게 분포 |
| GraphRAG | 약 15회 | 충분 — 후반부에 집중되나 제목/비교표에도 등장 |
| 지식 그래프 | 7회 | 양호 — GraphRAG 섹션에 집중, 타 섹션에서도 언급 |
| RAG 종류 | 1회 | 부족 — frontmatter에만 존재. 본문에 자연스럽게 추가 필요 |
| AI 검색 | 1회 | 부족 — 본문에서 거의 사용되지 않음 |

### 누락된 고가치 키워드

다음 키워드는 검색량이 있으나 본문에서 사용되지 않았거나 매우 적음:

1. **"RAG 뜻"** / **"RAG란"**: 초보자 검색 의도에 해당. 도입부 또는 첫 H2에 자연스럽게 삽입 권장. 예: "RAG란 무엇인지, RAG 뜻부터 짚어보겠습니다."
2. **"검색 증강 생성"**: RAG의 한국어 풀네임. 현재 1회 언급. H2 헤딩이나 첫 문단에서 1~2회 추가 사용 권장.
3. **"벡터 검색"** / **"벡터 데이터베이스"**: RAG 관련 기술 키워드. 기본 RAG 섹션에 이미 "벡터"가 언급되므로 "벡터 검색"으로 확장하면 좋음.
4. **"LLM 할루시네이션"** / **"AI 환각"**: RAG의 존재 이유와 직결. 도입부에 자연스럽게 삽입 가능. 예: "이른바 AI 환각(hallucination) 문제를 해결하려고 등장한 기술이 RAG입니다."
5. **"RAG 비교"**: 비교표 섹션 제목이나 본문에 삽입. 예: "세 가지 RAG 비교를 한눈에 정리하면..."
6. **"기업 AI 도입"** / **"사내 AI"**: AX 기업 블로그 특성상 B2B 검색어 포착에 유리.

### 조치 사항

- "RAG 종류"를 비교 섹션 도입부에 삽입: "RAG에도 여러 종류가 있습니다" → "RAG 종류를 정리하면..."
- "AI 검색"을 도입부 또는 결론에 1~2회 추가: "AI 검색 품질의 한계에 부딪히게 됩니다" (이미 유사한 문장 존재하므로 키워드만 조정)
- "RAG 뜻"을 첫 H2 근처에 삽입
- "AI 환각"을 도입부에 삽입

### 평가

- H1 1개, H2 7개, H3 7개로 계층 구조가 잘 잡혀 있음.
- H2에 "RAG", "Advanced RAG", "GraphRAG" 등 핵심 키워드가 자연스럽게 포함됨.
- H3 수준까지 적절히 세분화되어 Featured Snippet 노출 가능성 있음.

### 개선 권장

1. **"세 가지 RAG, 어떻게 다른가요?"** → **"RAG 종류별 비교: 기본 RAG vs Advanced RAG vs GraphRAG"** — "RAG 종류", "RAG 비교" 키워드를 헤딩에 포함시켜 검색 노출 강화.
2. **"어떤 RAG를 선택해야 할까요?"** → **"우리 기업에 맞는 RAG 선택 가이드"** — "기업", "RAG 선택" 키워드 포착 + B2B 검색 의도 대응.
3. **"현장에서 GraphRAG가 빛나는 세 가지 장면"** → **"GraphRAG 기업 활용 사례 3가지"** — "GraphRAG 활용 사례" 검색어 직접 포착.
