# 주간 콘텐츠 디자인 가이드

> 매주 월/수/금 발행하는 블로그 콘텐츠의 전략적 역할과 디자인 방향을 정의한다.

---

## 주간 콘텐츠 전략 개요

| 요일 | 콘텐츠 유형 | 전략적 역할 | 독자 심리 | 카테고리 |
|---|---|---|---|---|
| **월요일** | 신기술 트렌드 | 관심 끌기 | "이런 게 있었어?" | AI Trend |
| **수요일** | 문제 + 실행 | 공감과 필요성 일깨우기 | "우리도 이 문제 있는데..." | Thought Leadership |
| **금요일** | 실행 가이드형 | 신뢰 바탕 계약 전환 | "우리 회사에도 적용 가능하겠다" | Case Study |

> **핵심 차별화**: 최신 AI 기술을 눈높이에 맞게 풀어주는 전문성 있는 콘텐츠.
> 기술 설명(X) → 업무 적용 관점(O)

---

## 요일별 디자인 방향

### 월요일: 신기술 트렌드

**목표**: 호기심 유발 + "이거 우리 업무에도 쓸 수 있겠는데?" 반응 이끌기

**디자인 톤**:
- 시각적으로 임팩트 있는 썸네일 (미래지향적, 네온 컬러)
- 비교표(기존 vs 신기술) 필수 활용
- 기업 활용 사례 박스를 시각적으로 강조

**레이아웃 패턴**:
```
[썸네일 - 미래지향적 비주얼]
[리드 - 호기심 유발]
[기술 개념 - 간결하게]
[비교표 - 기존 방식 vs 새로운 방식]
[일러스트 1 - 핵심 변화 시각화]
[기업 활용 시나리오 - 실무 관점]
[일러스트 2 - 실무 적용 장면]
[AX 시사점 + CTA]
```

**컬러 키**: 네온 블루 + 퍼플 그라디언트 (혁신, 미래)
**타이포**: 제목에 질문형 or 숫자 포함 ("3가지 변화", "왜 주목해야 하는가")

---

### 수요일: 문제 + 실행

**목표**: 공감 형성 + "해결할 방법이 있구나" 인식 전환

**디자인 톤**:
- 문제 섹션은 진한 톤 (위기감, 긴장감)
- 해결 섹션은 밝은 톤으로 전환 (전환점 시각화)
- 실행 방법은 단계별 구조로 명확하게

**레이아웃 패턴**:
```
[썸네일 - 문제와 해결의 대비 비주얼]
[리드 - 공감 유도 (독자의 고민에서 시작)]
[문제 정의 - 구체적 상황 묘사]
[일러스트 1 - 문제 상황 시각화]
[왜 발생하는가 - 구조적 원인 분석]
[해결 방향 - 핵심 전환점 (인용 블록 강조)]
[일러스트 2 - 해결 구조 시각화]
[실행 방법 - 간결한 액션 가이드]
[CTA - 더 깊은 실행이 필요하다면]
```

**컬러 키**: 딥 블루 + 골드 (문제: 어두운 톤 → 해결: 밝은 골드로 전환)
**타이포**: 제목에 문제 키워드 + 해결 키워드 병치 ("~하는 이유 + 해결 방법")

---

### 금요일: 실행 가이드형 (사례 기반)

**목표**: "이 회사도 했으니 우리도 할 수 있다" 확신 + 계약 전환

**자료 원칙**:
- 금요일 `Case Study`의 주제 선정과 자료 수집은 `AX 훈련/` 폴더를 우선 사용한다.
- `AX 훈련/` 자료를 1차 근거로 삼고, 외부 검색은 맥락 보강이 필요할 때만 제한적으로 사용한다.
- `AX 훈련/` 자료는 AI 활용에는 비전문가이지만 자기 업무 맥락은 잘 아는 참여자들이 남긴 현장 기록으로 보고, AI 전문 해설보다 "이런 상황과 반응이 있었다"는 사례 관점으로 정리한다.
- 따라서 글의 톤은 권위적 AI 분석보다 관찰 기반의 실행 사례 소개에 가깝게 잡는다.

**디자인 톤**:
- Before/After 대비를 시각적으로 극대화
- 데이터/수치 강조 박스 활용
- 프로세스 다이어그램 필수

**레이아웃 패턴**:
```
[썸네일 - 전문적/신뢰감 있는 비주얼]
[리드 - 성과 수치로 시작 (결과 먼저)]
[산업 상황 - 해당 업종의 현실]
[기존 문제 - 구체적 페인포인트]
[일러스트 1 - 문제 → 솔루션 전환 시각화]
[AI 적용 방식 - 단계별 프로세스]
[다이어그램 - 솔루션 아키텍처/워크플로우]
[결과 Before/After - 비교표 필수]
[일러스트 2 - 성과/임팩트 시각화]
[시사점 + CTA - 강한 전환 유도]
```

**컬러 키**: 블루 + 화이트 (전문성, 신뢰)
**타이포**: 제목에 업종/성과가 드러나도록 ("건설사 입찰 자동화로 처리 시간 80% 단축")

## 요일 간 연결 전략

| 월요일 글 | → | 수요일 글 | → | 금요일 글 |
|---|---|---|---|---|
| "이런 기술이 나왔다" | | "이 기술 없으면 이런 문제가 생긴다" | | "이렇게 적용해서 이런 결과를 냈다" |
| 호기심 | | 필요성 | | 확신 |

> 같은 주의 콘텐츠가 하나의 스토리라인을 형성하면 이상적이나, 필수는 아니다.
> 주제가 연결되지 않더라도 각 요일의 전략적 역할은 반드시 지킨다.

## Company News (비정기)

Company News는 주간 정규 스케줄 외 비정기로 발행한다.
강의, 시상, 정부사업 수주 등 이벤트 발생 시 즉시 또는 당일 발행.
기존 `company-news.md` 템플릿을 그대로 사용한다.
