# Case Study 글 템플릿 (금요일: 실행 가이드형)

## 개요

- **전략적 역할**: 신뢰 바탕 계약 전환 — "우리 회사에도 적용 가능하겠다" 확신 유도
- **핵심 차별화**: 매직에꼴 교육/PoC 사례를 기반으로 제작. 실제 결과로 말하는 신뢰감
- **톤**: 객관적, 데이터 중심. Before/After로 변화를 체감하게
- **분량**: 1,800~2,500자

## 콘텐츠 구조 원칙

모든 변형에 공통 적용되는 4단 구조:

1. **산업 상황** — 해당 업종이 직면한 현실. 독자가 "우리 업계 얘기다" 느끼게
2. **기존 문제** — 기존 방식의 구체적 페인포인트
3. **AI 적용 방식** — 어떻게 해결했는지 프로세스 중심으로
4. **결과 (Before/After)** — 정량적 성과 비교. 이것이 핵심 설득 도구

> "이 회사도 했으니 우리도 할 수 있다"는 확신을 주는 것이 목표다.

## 디자인 요소

- **컬러 키**: 블루 + 화이트 (전문성, 신뢰)
- **썸네일**: 전문적이고 깔끔한 테크 비주얼
- **Before/After 비교표**: 모든 변형에서 필수. 성과를 시각적으로 극대화
- **프로세스 다이어그램**: 솔루션 아키텍처/워크플로우 시각화 권장
- **데이터 강조 박스**: 핵심 수치는 인용 블록으로 강조
- **제목 패턴**: em-dash(—)로 부제 연결. "업종 + 핵심 성과 — 구체적 변화"
  - 예: "건설사 입찰 자동화 — AI 파이프라인으로 처리 시간 80% 단축"

## 글쓰기 스타일 규칙

### 제목 포맷
- **H1(글 제목)**: `{업종/대상} {핵심 키워드} — {구체적 성과/변화}` 형태
- **H2(섹션 제목)**: `{주제} — {한 줄 설명}` 형태. 읽지 않아도 섹션 내용 파악 가능
  - 예: "기존 방식의 한계 — 수작업이 만든 3일의 병목"
- **H3(하위 항목)**: 프로세스 단계나 교훈 항목

### 리드 문단 (2단락 구조)
- **1단락 (훅)**: 결과 먼저 제시하거나 독자의 공감 상황에서 시작. 3~4문장.
  - 예: "수작업으로 3일 걸리던 데이터 분석. AI 파이프라인 도입 후 2시간으로 줄었습니다."
- **2단락 (컨텍스트)**: 이 사례의 맥락과 의미를 설명. 2~3문장.

### 본문 강조 패턴
- **볼드 키프레이즈**: 핵심 개념/수치가 처음 등장할 때 **볼드** 처리
- **인라인 링크**: 관련 기술/도구 언급 시 공식 페이지로 하이퍼링크 연결
- **핵심 요약 블록**: 섹션 끝에 `> **핵심**: ...` 형태로 한 줄 요약
- **독자 참여 질문**: 2~3개 섹션에 "여러분의 조직에서는 ~?" 형태로 분산 배치

## 구조 변형 선택 가이드

글마다 다른 느낌을 주기 위해, 사례 성격에 맞는 구조를 선택한다. **매번 같은 구조를 반복하지 않는다.**

| 변형 | 적합한 사례 | 느낌 |
|---|---|---|
| **A. 성과 중심형** (기본) | 정량 성과가 뚜렷한 사례 | "숫자로 증명된다" — 결과 우선 |
| **B. 여정 중심형** | 도입 과정에서의 시행착오/학습이 풍부한 사례 | "이 과정이 도움이 된다" — 프로세스 공유 |

## 작성 주의사항

- **Before/After 비교표 필수**: 모든 변형에서 성과 비교표를 반드시 포함한다. 이것이 신뢰 전환의 핵심 도구.
- **업종 특정성**: 리드 문단에서 업종을 명시한다. "한 기업"이 아니라 "한 건설기업", "한 제조 R&D팀" 등 구체적으로.
- **em-dash 제목 패턴**: H1, H2 제목에 "— 부제" 형태를 사용하여 섹션 내용을 예고한다.
- **2단락 리드 구조**: 훅(결과/공감) + 컨텍스트(배경/의미)로 구성한다.
- **볼드 키프레이즈**: 핵심 수치/개념 첫 등장 시 **볼드** 처리. 문단당 1~2개로 절제.
- **인라인 링크**: 관련 기술/도구/서비스는 공식 페이지로 하이퍼링크.
- **AX 시사점 H3 분리**: "AX 관점에서~" 섹션은 2~3개 H3 교훈으로 나눠 깊이를 더한다.
- **독립 마무리 섹션**: 별도 H2로 핵심 메시지 + 성찰 질문 + 행동 유도.
- **CTA는 링크 형태**: `[매직에꼴 AX 컨설팅 알아보기 →](URL)` — 블록쿼트 없이.
- 수치는 반드시 소스에 있는 것만 사용. 임의 추정/반올림 금지.
- 고객사 NDA 정보는 익명화하거나 삭제.
- 성과 수치가 없으면 "초기 피드백 기반 정성적 평가"로 대체.
- 경쟁사나 특정 기업 비판은 피한다.
- **직전 포스트와 같은 구조 변형을 연속 사용하지 않는다.** 다양성이 핵심.
